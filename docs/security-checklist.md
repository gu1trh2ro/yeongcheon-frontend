# Security Checklist

## API Key

- OpenAI API Key는 프론트에 넣지 않는다.
- Claude API Key는 프론트에 넣지 않는다.
- 공공데이터 serviceKey는 프론트에 넣지 않는다.
- 백엔드 Secret Key는 프론트에 넣지 않는다.
- 네이버 지도 Client ID는 프론트에서 사용할 수 있지만 `.env`에서 관리한다.
- `.env`는 GitHub에 올리지 않는다.
- `.env.example`만 올린다.

## 실종자 데이터

- 실제 실종자 이름, 얼굴, 신체 특징을 기본 화면에 직접 노출하지 않는다.
- 시연에서는 가상 실종자 데이터를 사용한다.
- 화면에는 “실종자 후보”, “유사도”, “담당자 확인 필요” 중심으로 표시한다.
- “실종자 확정”이라는 표현을 사용하지 않는다.

## 실종자 후보 처리

- 실제 자동 신고나 신고서 작성 기능처럼 보이게 만들지 않는다.
- 버튼 문구는 “담당자 검토 표시”, “상세 내용 확인”처럼 사람의 검토 흐름으로 사용한다.
- “경찰 자동 신고 완료”, “신고 초안 생성”, “모의 신고” 문구는 사용하지 않는다.

## 프론트 입력값

- 사용자 입력값을 HTML로 직접 렌더링하지 않는다.
- dangerouslySetInnerHTML을 사용하지 않는다.
- memo 입력값은 빈 값 방지와 길이 제한을 적용한다.
- eventId, robotId는 백엔드 전송 전 기본 검증을 한다.

## 배포

- 배포 시 HTTPS 사용
- 백엔드 CORS는 프론트 배포 도메인만 허용
